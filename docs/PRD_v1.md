# NHViewer PRD v1

## 1. 背景与目标

### 1.1 背景
现有实现在可维护性、模块边界和可回归性上存在持续性风险，导致功能迭代慢、问题定位成本高、易回归。

### 1.2 产品目标
在保留 EHViewer 交互习惯与核心体验的前提下，完成可维护架构重建，并优先覆盖首发核心流程。

### 1.3 成功指标（v1）
- 核心路径可用：首页 -> 搜索/筛选 -> 详情 -> 阅读 -> 收藏/历史可闭环。
- 核心页面启动与切换稳定，无阻断级崩溃。
- 阅读进度可自动记忆并可快速继续。
- 本地收藏与历史 CRUD 正常，列表渲染稳定。

## 2. 目标用户与典型场景
- 高频浏览用户：快速发现内容，依赖热门与筛选。
- 目标检索用户：通过关键词 + 标签精准定位。
- 连续阅读用户：依赖进度记忆与快速续读。
- 归档用户：重视下载队列、失败重试和目录管理。

## 3. 功能范围

### 3.1 首发必须（Phase 1）
- 首页/热门
- 搜索（关键词、分页、排序）
- 标签过滤
- 详情页（信息、标签、页数、封面/缩略图）
- 阅读器（翻页、缩放、预加载、横竖屏、进度记忆）
- 收藏（本地）
- 历史（本地）
- 设置中心（图片质量、缓存、并发、主题、语言）

### 3.2 二期（Phase 2）
- EH 风格细节：长按菜单、批量操作、快速筛选、返回位置记忆
- 深度性能优化：低端机降级、分页预取策略增强
- 可用性增强：弱网容错、离线提示体系化

### 3.3 并行模块（可与 Phase 1/2 并行）
- 下载与文件系统：队列、暂停/恢复、重试、通知栏进度、去重、媒体扫描

## 4. 非功能需求
- 架构：data/domain/ui 三层，接口先行，依赖单向流动。
- 状态：ViewModel + StateFlow，统一加载/空态/错误态模型。
- 数据：Room + DataStore，离线状态可恢复。
- 网络：Retrofit + OkHttp + Kotlinx Serialization，统一错误映射。
- 可测性：Repository 与 UseCase 可独立单测。

## 5. 用户旅程（核心）
1. 进入首页查看热门与推荐。
2. 通过关键词与标签筛选查找目标画廊。
3. 进入详情页预览并判断是否阅读/收藏/下载。
4. 进入阅读器阅读并自动保存进度。
5. 在历史中快速续读，或在收藏中管理常看内容。

## 6. 验收标准（v1）
- 搜索结果可分页加载，标签筛选可叠加。
- 详情页完整展示关键元数据，缩略图可点击进入阅读。
- 阅读器重进后恢复到上次页码。
- 收藏/历史支持新增、删除、读取、排序。
- 设置修改后即时生效并持久化。

## 7. 里程碑与排期（建议）
- D1-D2：范围定义、信息架构、PRD 与线框。
- D3-D5：基础技术架构搭建（网络、存储、状态层）。
- W2-W3：首发核心能力落地（列表/搜索、详情、阅读、收藏历史、设置）。
- W4：EH 风格体验对齐 + 下载能力完善 + 质量收口。

## 8. 风险与约束
- API 不稳定或限流：通过重试策略与错误分级降低影响。
- 图片加载高内存占用：增加预加载窗口和降级策略。
- 旧模块替换风险：采用兼容层和模块渐进替换。

## 9. 重构落地策略（强约束）
- 不在旧代码上继续补洞，以新模块逐步替换旧页面。
- 每替换一个模块，必须达到“可编译、可运行、可回归”。
- 保留兼容层，直到新模块覆盖完成后再统一删除旧实现。
