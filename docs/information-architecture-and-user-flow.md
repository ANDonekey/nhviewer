# 信息架构与页面流

## 1. 信息架构（IA）

- 首页/热门
- 搜索
- 标签筛选
- 详情
- 阅读器
- 下载
- 收藏
- 历史
- 设置

## 2. 导航结构（首发）

- 底部一级导航（建议）
- 首页
- 搜索
- 收藏
- 历史
- 设置

详情、阅读器、下载管理作为二级/全屏页进入。

## 3. 页面流（首发核心）

`mermaid
flowchart TD
    A["首页/热门"] --> B["搜索页"]
    B --> C["标签筛选面板"]
    B --> D["结果列表"]
    D --> E["详情页"]
    E --> F["预览缩略图"]
    E --> G["阅读器"]
    E --> H["收藏操作"]
    G --> I["阅读进度保存"]
    I --> J["历史列表"]
    J --> G
    E --> K["下载任务"]
    L["设置"] --> G
    L --> D
`

## 4. 首发与二期边界

### 4.1 首发必须
- 首页热门浏览
- 搜索 + 标签筛选 + 分页
- 详情 + 预览
- 阅读器与进度记忆
- 收藏/历史本地管理
- 设置中心

### 4.2 二期
- 长按菜单与批量操作
- 快速筛选入口优化
- 返回位置记忆（跨页面）
- 更激进的性能策略与弱网容错增强

## 5. 状态流约束（跨页面）
- 搜索参数、排序、筛选条件可恢复。
- 详情返回列表时记忆滚动位置。
- 阅读器退出后，详情与历史页可读取最新阅读位置。
- 设置变更通过统一配置流广播到相关页面。
